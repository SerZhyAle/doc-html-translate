ОПРОСНИК ПЕРЕД СТАРТОМ РАЗРАБОТКИ EPUB2HTML

Q1) Политика существующего выходного каталога
Если каталог книги уже существует, что делать?
A) сразу открывать готовый HTML
B) всегда перегенерировать
C) перегенерировать только с флагом force
Ответ: Сразу открывать готовый

Q2) Политика ошибок перевода
Если Google API недоступен/квота исчерпана:
A) завершать с ошибкой
B) открывать непереведённый результат
C) частично переводить, что удалось, и открывать
Ответ: Сообщать об ошибке ставить прогресс на паузу, чтобы пользователь увидел об этом сообщение в открывшемся окне shell.

Q3) Хранение API ключа
Как храним ключ в v1?
A) hardcoded (временно)
B) env-переменная
C) локальный конфиг-файл
D) смешанный режим (приоритеты)
Ответ: Я хочу харткодед.

Q4) Регистрация ассоциации EPUB
Подтвердить режим:
A) только HKCU (текущий пользователь)
B) добавить опционально HKLM (админ режим)
Ответ: Я не понимаю, зачем админ режим, если он будет работать для текущего пользователя.

Q5) Браузер для открытия результата
A) браузер по умолчанию Windows
B) строго Chrome
C) configurable флагом
Ответ: Браузер по умолчанию.

Q6) Консольное окно
После завершения:
A) закрывать автоматически
B) оставлять открытым
C) флаг close-on-complete
Ответ: Если все совершилось успешно, закрывать автоматически.

Q7) Поддержка «грязных» EPUB
A) fail-fast (строго)
B) best-effort (пропуск проблемных частей с предупреждением)
Ответ: Пропуск проблемных частей.

Q8) Масштаб производительности
Ожидаемый максимум книги для v1:
A) до 5 МБ
B) до 20 МБ
C) 20+ МБ (нужна оптимизация и ограничения)
Ответ: До двадцати мегабайт. Там могут быть картинки. Это нормально, их легко достать и положить.

Q9) Ограничения перевода
Что точно не переводим:
A) script/style
B) code/pre/math тоже не переводить
C) переводить всё текстовое, кроме служебного
Ответ: Ну а штам-страница должна быть как это Кипаб. То есть это заголовки, Страницы. Все как в оригинале. Конечно, если у нас там шрифта какого-то нет и так далее, надо заменять на существующее.

Q10) Языковые дефолты
Подтвердить: src=auto, dst=ru
Да / Нет (укажите свои)
Ответ: По умолчанию это с английского на русский. Все остальное с помощью флага.

Q11) Набор тестовых EPUB для приемки
Сможете дать 3–5 реальных файлов для regression?
Да / Нет
Если да, укажите типы: epub2, epub3, с картинками, со шрифтами, «проблемный».
Ответ: Да, я предоставлю файлы, правда не знаю их версии. Предоставлю с картинками без картинок. Положу здесь каталог Test_epub

Q12) Критерий «шаг выполнен» для коммита
Что считать «удачным шагом» (после которого обязателен коммит)?
A) только успешная сборка
B) сборка + тесты
C) сборка + тесты + lint
Ответ: Если в программе еще нет возможности сборки, то и комиты ни о чем не нужны. Нам нужны комиты только для роллбека при необходимости.

Q13) Приоритет первой рабочей версии
A) сначала без перевода (конвертация + открытие)
B) сразу с переводом в MVP
C) сначала регистрация/UX, потом конвертация
Ответ: Да, первая рабочая версия может быть без перевода. Проверим начало конвертации HTML. Я думаю, такие вещи, как регистрация Виндо, можно даже не проверять, они Элементарны'. 

Q14) Формат логирования
A) минимальный (только шаги и ошибки)
B) подробный (тайминги, файлы, стадии)
C) оба режима через флаг verbose
Ответ: Ты имеешь в виду логирование результатов программы? Они меня не интересуют. Только на стадии дебага программы здесь. В релизной версии мне не нужно логирование никуда во внешние фазы.

Q15) Ограничения по безопасности в v1
Допустимо ли временно: hardcoded key + локальный запуск без шифрования секрета?
Да / Нет
Если нет — какой обязательный минимум защиты сразу?
Ответ: Можно снять все ограничения по безопасности. Это программа для меня. На моем компьютере. Пользуюсь только я и все такое.
