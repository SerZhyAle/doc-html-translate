## Overview

CLI-утилита на Go. Нет GUI. При запуске с флагом `-register` регистрируется в Windows как обработчик `.epub`-файлов. При открытии `.epub` через проводник — конвертирует книгу в HTML, переводит на русский через Google Translate API, открывает в браузере по умолчанию.

---

## Functional Requirements

### 1. Windows Registration
- Флаг: `-register`
- Регистрирует программу как обработчик `.epub` в **HKCU** (без прав администратора).
- Вывод в консоль: `Registered as default .epub handler (HKCU)`.
- Регистрация: `HKCU\Software\Classes\.epub` + `HKCU\Software\Classes\epub2html\shell\open\command`

### 2. Core Workflow (при вызове `epub2html.exe <file.epub>`)

```
[1/4] Extracting EPUB...
[2/4] Building HTML structure...
[3/4] Translating (N pages)... [##########] 100%
[4/4] Opening in browser...
Done.
```

**Шаги:**

1. **Extract**: распаковать `.epub` (это ZIP) в подкаталог `<epub_dir>/<epub_name>/`
   - Структура — как определено в EPUB `spine` (порядок глав из `content.opf`)
   - Скопировать все assets: изображения, шрифты, CSS
   - EPUB 2 и EPUB 3 — оба поддерживаются

2. **HTML**: сгенерировать `index.html` в корне подкаталога
   - Структура и навигация — по `spine` из EPUB (не изменять, не агрегировать)

3. **Translate**: перевести текстовое содержимое HTML-страниц через Google Translate API
   - API: Cloud Translation API v2 (REST, API Key)
   - API Key: **hardcoded** в бинаре (личный инструмент, не для дистрибуции)
   - Target language: `ru` (по умолчанию, переопределяется флагом `-dst`)
   - Source language: `auto` (по умолчанию, переопределяется флагом `-src`)
   - Переводить только `innerText` элементов, не трогать атрибуты/ссылки/теги

4. **Open**: открыть `index.html` через `ShellExecute` (браузер по умолчанию Windows)

### 3. Re-open Logic (подкаталог уже существует)
- Пропустить шаги 1–3 (extraction + translation).
- Сразу открыть `index.html` в браузере.

### 4. Progress / UX
- Программа запускается в консольном окне (стандартное Windows `.exe`).
- Вывод: текстовые шаги `[N/M] Description...` + прогресс-бар при переводе `[##########] 100%`.

## CLI Flags

| Флаг            | Описание                                              | Пример                               |
|-----------------|-------------------------------------------------------|--------------------------------------|
| `-register`     | Зарегистрировать как обработчик .epub в HKCU          | `epub2html.exe -register`            |
| `-no-translate` | Пропустить шаг перевода                               | `epub2html.exe book.epub -no-translate` |
| `-src <lang>`   | Исходный язык (по умолчанию: `auto`)                  | `-src en`                            |
| `-dst <lang>`   | Целевой язык перевода (по умолчанию: `ru`)            | `-dst de`                            |

Стиль флагов: короткий (`-flag`, `-flag value`), стандарт Go `flag` package.

## Technical Stack

| Компонент          | Решение                                        |
|--------------------|------------------------------------------------|
| Language           | Go 1.21+                                       |
| EPUB parsing       | `archive/zip` + `encoding/xml`                 |
| HTML manipulation  | `golang.org/x/net/html`                        |
| Google Translate   | REST API v2, `net/http`, hardcoded API Key     |
| Windows Registry   | `golang.org/x/sys/windows/registry`            |
| Browser open       | `ShellExecute` via `golang.org/x/sys/windows`  |
| Progress bar       | Ручной рендер в `os.Stdout`                    |
